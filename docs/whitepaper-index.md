# Whitepaper Guide

The whitepaper is the long-form argument behind Lucidity-Bench. The repository pages are the operational layer.

## What the Whitepaper Argues

The core claim is that better agents are not simply better at generating longer traces. They are better at noticing conflict, exiting unproductive local search, and escalating to a more appropriate reasoning level.

## Who Should Read It

This is most useful for:

- agent infrastructure builders,
- benchmark designers,
- researchers thinking about control policies rather than only model scale,
- readers who want the architectural thesis behind LEI, DEE, and DEOA.

## How It Connects to This Repo

- the whitepaper explains the thesis,
- the docs define the measurement contract,
- the cases make the failure patterns public,
- the benchmark task packs create repeatable pressure.

## Reading Path

If you are new here:

1. read [`methodology.md`](methodology.md),
2. skim [`metrics.md`](metrics.md),
3. read the first public case,
4. then return to the full whitepaper.
